2. 各维度详细调研指南

D01：业务战略与数字化转型愿景

🎯 本维度核心目标：理解企业未来3-5年业务战略方向、数字化转型优先级、关键业务痛点，确保后续所有技术调研和方案设计始终与业务价值对齐（Business-IT Alignment）。

📋 必须收集的关键问题

#问题收集目的
1企业未来3-5年核心业务战略是什么？（如全球化扩张、智能制造、产品数字化、供应链优化）确定IT投资方向
2当前最影响业务发展的IT瓶颈/痛点TOP 3是什么？定义优先级
3数字化转型的关键KPI是什么？（如OEE提升、供应链可视化率、客户数字体验NPS）量化价值
4是否有明确的IT预算周期和3年投资规划？IT支出占营收比例？方案可行性约束
5是否有集团级IT标准化要求？总部与工厂的IT管控模式？（集中/联邦/混合）架构治理模式
6近期是否有并购/新建工厂/产线搬迁等重大变化？时间约束
7核心业务对IT的可用性要求？是否有明确的SLA？BCDR设计输入
8企业对AI/大模型的战略定位？是已有明确场景还是探索阶段？AI算力规划输入

🔧 推荐调研工具/手段

  • CxO访谈（CIO/CDO/CFO/COO，1对1或工作坊形式，每场60-90分钟）
  • 业务流程工作坊（Value Stream Mapping，识别数字化触点）
  • Cisco Business Critical Services (BCS) 评估框架
  • TOGAF Business Architecture 方法论
  • 已有的IT战略规划文档/咨询报告收集

⚠️ 常见痛点 & Cisco方案提示

痛点Cisco关联方案
业务部门与IT部门对优先级认知不一致Cisco CX（Customer Experience）业务成果工作坊
IT投资缺乏量化回报衡量Cisco Business Case / ROI工具
多工厂/多区域标准化困难Cisco DNA/Catalyst Center统一策略管理 + SD-WAN全球模板下发

D02：业务应用与核心系统

🎯 本维度核心目标：完整梳理业务应用全景图（Application Portfolio），理解应用间依赖关系、部署模式、性能要求，为后续基础设施设计提供"负载画像"。

📋 必须收集的关键问题

#问题收集目的
1完整的业务应用清单？（ERP、MES、PLM、WMS、SCM、CRM、OA、HR等），每个应用的厂商/版本/部署位置？应用全景图
2哪些是Tier-1关键业务应用（停机即停产/停业）？RTO/RPO要求？可用性设计
3应用间的调用关系和数据流向？（如ERP↔MES↔SCADA的实时交互）网络流量建模
4各应用的并发用户数、峰值TPS、响应时间要求？性能基线
5应用的云化/容器化现状？（传统VM / 容器化 / SaaS）未来云化路线图？混合云架构设计
6是否有自研应用？开发团队的DevOps成熟度？CI/CD工具链？运维自动化
7当前应用性能监控手段？是否有APM工具？可观测性差距
8近12个月最严重的应用故障事件？根因是什么？痛点挖掘

🔧 推荐调研工具/手段

  • AppDynamics — 自动发现应用拓扑、依赖关系、性能基线（可申请POC部署）
  • ThousandEyes — 对SaaS应用（如Office 365、Salesforce）的端到端路径质量评估
  • Cisco Intersight — 收集计算资源利用率与应用映射
  • CMDB导出（ServiceNow / BMC）
  • 应用负责人访谈（每个Tier-1应用至少1场）

⚠️ 常见痛点 & Cisco方案提示

痛点Cisco关联方案
应用间依赖关系不清，故障排查难AppDynamics Flow Maps + Business iQ
SaaS应用体验差（如Teams/Webex卡顿）但无法定位ThousandEyes 端到端Internet路径可视化
传统应用上云缺乏评估方法Cisco Cloud Assessment + Intersight Workload Optimizer（IWO）
应用在多数据中心间的流量调度不合理ACI Multi-Site + SD-WAN 应用感知路由

D03：数据治理与数据平台

🎯 本维度核心目标：了解企业数据资产全貌、数据架构现状、数据质量与合规状态，为AI/大数据平台和数据安全方案提供基础。

📋 必须收集的关键问题

#问题收集目的
1企业核心数据资产有哪些？（生产数据、客户数据、供应链数据、财务数据）存储在哪里？数据地图
2是否有数据湖/数据仓库？技术栈是什么？（Hadoop/Spark/Snowflake/阿里云MaxCompute等）数据架构现状
3数据分类分级是否已建立？敏感数据（个人信息、商业秘密）如何标识和管控？数据安全合规
4数据在各系统间的流转方式？（ETL/ELT/实时流/API）数据集成工具链？数据管道
5数据质量如何？是否有数据清洗/校验机制？AI数据就绪度
6是否有数据治理组织（如数据治理委员会、数据所有者制度）？治理成熟度
7跨境数据传输需求？（如中国工厂数据传至海外总部）合规约束
8大数据/AI场景的数据量级？增长趋势？（TB/PB级？年增长率？）存储/算力规划

🔧 推荐调研工具/手段

  • 数据架构文档收集 + 数据库管理员访谈
  • AppDynamics Database Visibility — 数据库性能与依赖分析
  • Cisco Secure Network Analytics (Stealthwatch) — 数据流行为分析
  • 现有数据治理平台截屏/报告收集
  • 等保2.0/GDPR/PIPL合规检查表比对

⚠️ 常见痛点 & Cisco方案提示

痛点Cisco关联方案
数据孤岛严重，各系统数据无法打通Cisco + 生态合作伙伴数据集成（API Gateway / Service Mesh）
敏感数据流向不可见Cisco Secure Network Analytics 数据流可视化 + ISE 基于身份的访问控制
跨境数据合规风险SD-WAN + 加密隧道 + 合规路由策略
AI模型训练数据散落在多个系统数据平台统一设计 + 高性能存储网络（NVMe-oF / RDMA）

D04：数据中心网络

🎯 本维度核心目标：全面了解数据中心网络架构（Underlay + Overlay）、DCI方案、云互联现状、网络性能与可靠性，识别架构演进方向。

📋 必须收集的关键问题

#问题收集目的
1数据中心数量、位置、定位（主DC/灾备DC/边缘DC）？Tier等级？DC全景
2当前DC网络架构？（传统三层 / Spine-Leaf / Fabric）Underlay协议？架构评估
3是否部署了Overlay？（VXLAN / ACI / NSX / 其他）Fabric管理工具？自动化能力
4DCI方案？（Dark Fiber / DWDM / OTV / VXLAN Multi-Site / EVPN）带宽和延迟？多DC互联
5公有云互联方式？（阿里云Express Connect / AWS Direct Connect / Azure ExpressRoute）带宽？混合云网络
6DC内东西向流量占比？微隔离需求？当前是否有策略控制？安全需求
7网络设备型号、软件版本、生命周期状态（EoL/EoS）？刷新规划
8近12个月DC网络重大故障？MTTR？根因？痛点

🔧 推荐调研工具/手段

  • Cisco ACI Topology导出 / DCNM/NDFC Dashboard截屏（如已部署）
  • NetFlow / sFlow采集分析（东西向/南北向流量画像）
  • Catalyst Center / DNA Center设备发现（如果是Cisco设备）
  • SNMP/CLI方式收集设备清单（show inventory, show version）
  • 现场机房巡检（物理拓扑、布线、标签、电力冗余）
  • Cisco Network Assurance Engine (NAE) — ACI环境下的策略一致性验证

⚠️ 常见痛点 & Cisco方案提示

痛点Cisco关联方案
传统三层架构，VLAN泛滥，变更慢Cisco ACI — 基于策略的自动化Fabric（Intent-Based）
多DC策略一致性难保证ACI Multi-Site Orchestrator (Nexus Dashboard Orchestrator)
云互联带宽不足/延迟高Cisco SD-WAN Cloud OnRamp + 专线优化
东西向安全盲区ACI微隔离 + Secure Workload (Tetration)
设备老化，EoL风险高Cisco Nexus 9000系列刷新 + Smart Net Total Care EoL报告

D05：计算资源与云平台

🎯 本维度核心目标：摸清计算资源现状（物理/虚拟/容器/云），评估利用率、云化程度、混合云治理能力，为资源优化和云战略提供数据支撑。

📋 必须收集的关键问题

#问题收集目的
1物理服务器清单？（品牌/型号/CPU/内存/年限）Cisco UCS占比？资产盘点
2虚拟化平台？（VMware vSphere / Hyper-V / KVM / 华为FusionCompute）版本和License模式？虚拟化评估
3超融合(HCI)部署情况？（HyperFlex / Nutanix / vSAN / SmartX）规模？HCI现状
4容器平台？（K8s / OpenShift / Rancher / ACK）规模和成熟度？云原生评估
5公有云使用情况？（阿里云/AWS/Azure/华为云）IaaS/PaaS/SaaS各有多少工作负载？月度云支出？云成本优化
6混合云管理平台？（Cisco Intersight / VMware Aria / 原生工具）统一性如何？管理能力
7整体计算利用率？（CPU/Memory平均和峰值）是否存在大量zombie VM？资源优化
8未来2年是否有计算资源扩容或迁移计划？（如VMware收购后的战略调整？）趋势

🔧 推荐调研工具/手段

  • Cisco Intersight — UCS/HyperFlex资产发现、利用率监控（可做免费评估）
  • Intersight Workload Optimizer (IWO) — 智能工作负载优化建议
  • vCenter / SCVMM Dashboard导出
  • 云账单分析（阿里云费用中心 / AWS Cost Explorer）
  • 容器平台Dashboard（kubectl top / Prometheus + Grafana）

⚠️ 常见痛点 & Cisco方案提示

痛点Cisco关联方案
服务器老旧，运维成本高Cisco UCS X-Series — 模块化、面向未来的计算平台
VMware License成本激增（被Broadcom收购后）Cisco HyperFlex（内置虚拟化）/ 评估替代虚拟化方案
多云管理碎片化Cisco Intersight — 统一混合云运维管理平面
容器平台缺乏网络/安全策略Cisco Calico / Isovalent (eBPF) 容器网络 + 微隔离
云成本失控IWO + 云FinOps实践
